# Support

Need help with MailVoyage? Use the right channel so your request gets handled faster.

## Where to Ask

### 1) Questions and setup help
Use GitHub Discussions:
- https://github.com/navaranjithsai/MailVoyage/discussions

Use this for:
- installation or environment setup help
- provider setup questions (Gmail, Outlook, Yahoo, custom IMAP/SMTP)
- usage questions and troubleshooting
- architecture or roadmap discussions

### 2) Bugs and reproducible problems
Open a GitHub Issue using the Bug Report template.

Use this for:
- crashes
- incorrect behavior
- regression after an update
- API/UI behavior that does not match docs

### 3) Feature ideas and improvements
Open a GitHub Issue using the Feature Request template.

Use this for:
- new features
- UX improvements
- performance improvements
- developer workflow suggestions

### 4) Documentation updates
Open a GitHub Issue using the Documentation template.

Use this for:
- unclear README/wiki sections
- incorrect setup steps
- missing examples/screenshots

### 5) Security vulnerabilities
Do not open public issues.
Use private reporting only:
- https://github.com/navaranjithsai/MailVoyage/security/advisories/new

## Before Opening an Issue

1. Check the README and wiki first.
2. Search existing issues/discussions to avoid duplicates.
3. Include clear reproduction steps and expected vs actual behavior.
4. Include logs/screenshots when helpful.

## Privacy and Safety

- Never post secrets, tokens, passwords, API keys, or `.env` values.
- Redact personal data from screenshots and logs before posting.

Thanks for helping improve MailVoyage.
